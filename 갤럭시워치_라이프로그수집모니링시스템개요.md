## Ⅰ. 시스템 개요

**명칭** : 스마트워치 기반 라이프로그 데이터 수집·관리 솔루션 개발

재택환자에게 할당된 갤럭시워치를 통하여 환자의 생체신호, 활동, 운동, 수면 데이터를 수집하여 의료진이 환자의 상태를 모니터링하고 임상적 연구 데이터로 활용하기 위한 시스템

### 프로젝트목표

* 스마트워치(갤럭시워치)를 통해 스마트폰으로 수집되는 라이프로그 데이터를 수집·전송·관리하는 솔루션을 구축
* 연구 참여자 또는 대상자로부터 수집되는 활동, 수면, 심박수 등 생체·행동 데이터를 서버로 전송·저장·관리할 수 있는 체계를 마련하여, 향후 디지털 바이오마커 연구 및 임상실증 수행에 활용 가능한 기반 시스템을 확보한다.

## Ⅱ. 상세내용

### 주요 내용

* 갤럭시워치 라이프로그 데이터의 동기화 및 서버로 전송하는 안드로이드 기반 스마트폰 앱 개발
* 백엔드 서버 구축 및 데이터 저장·관리 기능 구현
* 수집 모니터링 웹 구축 (수집 상태 확인 및 CSV 다운로드)
* 대상자-장치 매핑 관리 기능 구현

### 기본 방향

* 갤럭시워치의 데이터의 연동 방식을 통해 스마트워치 건강 데이터를 수집한다.
* 대상자 앱, 서버, 수집 모니터링 웹이 연계되는 구조로 구성한다.
* 수집된 데이터는 CSV 형식으로 다운로드 가능하도록 구현한다.

### 기대효과

* 스마트워치 기반 라이프로그 수집체계를 통해 디지털 헬스 연구 수행 기반을 마련
* 수집 → 전송·저장 → 다운로드로 이어지는 기본 데이터 파이프라인을 확보
* 향후 복약 모니터링, 증상 추적, 디지털 바이오마커 탐색 등 다양한 연구로 확장할 수 있는 기반을 확보

## Ⅲ. 범위 및 내용

### 1\. 시스템 전체 구성

본 과업은 다음의 3개 영역을 포함하여 수행한다. 각 구성요소는 상호 연계되어 스마트워치-스마트폰--서버-수집 모니터링 웹으로 이어지는 전체 데이터 흐름이 정상적으로 작동하도록 구현

* 안드로이드 앱(수집단)
* 백엔드 서버(데이터단)
* 수집 모니터링 웹(수집 상태 확인 및 다운로드 전용)

### 2\. 안드로이드 앱 개발

#### 2.1 기본 기능

* 안드로이드 기반 모바일 앱을 개발한다.
* 사용자는 앱 설치 후 로그인하여 본인 데이터 수집 상태를 확인할 수 있다.
* 앱은 갤럭시워치 데이터 동기화하여 데이터 전송 수행, 전송 결과 확인 기능을 제공한다.
* 앱은 백그라운드 기반의 주기적 전송 구조를 포함하며, 전송 실패 시 재시도 또는 오류 상태를 표시한다.

#### 3.2 사용자 관리

* 식별번호(가명정보) 기반의 로그인·로그아웃 기능을 제공한다.
* 사용자 기본정보 조회 및 수정 기능(가명정보 범위 내)

#### 3.3 사용자 편의 기능

* 최근 전송 시각 표시
* 수집 항목별 수집 상태 표시
* 전송 및 수집 오류 상태표시 기능

### 라이프로그 데이터 수집 및 연동

#### 4.1 연동 방식

* 스마트워치에서 수집된 건강데이터를 별도로 제작된 앱에서 수집하여 서버로 전송한다.
* **기본 데이터 흐름**: 워치 → 앱 → 서버

#### 4.2 기본 수집 대상 항목

수집대상 항목은 아래의 데이터 기본으로 하된 기기가 제공하는 범위에서 조정하여 확정한다.

* 걸음 수
* 소모 칼로리
* 활동 시간
* 운동 기록
* 이동 거리
* 수면 세션
* 수면 단계
* 심박수
* 안정 시 심박수
* 산소포화도

### 5\. 백엔드 서버 개발

#### 5.1 서버 기능

* 모바일 앱으로부터 라이프로그 데이터를 수집할 수 있는 API 개발
* 사용자 인증(Authentication) 및 인가(Authorization) 기능 구현 (로그인 여부 및 접근 허용 여부 판단 수준)
* 데이터 저장, 조회, 수정, 삭제 기능 구현
* 수집 이력 관리 및 로그 관리 기능 구현
* 관리자용 조회 및 CSV 다운로드 기능 제공

#### 5.2 데이터 저장 구조

* 시계열 데이터 저장에 적합한 구조로 설계하여야 한다.
* 사용자 정보, 장치 정보, 라이프로그 데이터, 수집 이력 등을 구분하여 관리하여야 한다.
* 데이터 무결성과 수집 이력을 통한 추적 가능성을 확보할 수 있어야 한다.

### 6\. 모니터링 웹 구축

본 과업의 웹 구성요소는 수집 데이터의 상태 확인 및 CSV 다운로드를 위한 기능으로 구성한다.

#### 6.1 수집 상태 확인 기능

* 대상자 목록 조회 (식별번호 및 수집 활성 여부 수준)
* 장치 목록 조회
* 대상자-장치 매핑 현황 조회
* 라이프로그 수집 이력 조회
* 항목별/대상자별/기간별 데이터 조회
* 수집 실패 또는 전송 오류 건 확인

#### 6.2 데이터 다운로드 기능

* 기간 및 대상자 식별번호 선택을 통한 CSV 형식 데이터 다운로드
* 다운로드 파일은 항목명, 측정시각, 값, 대상자 식별번호(가명정보) 등이 정리된 구조로 제공한다.

#### 6.3 기본 관리 기능

* 대상자 등록/수정/비활성화 기능 (식별번호 기반)
* 장치 등록/수정/상태관리 기능
* 대상자-장치 배정 및 해제
* 관리자 로그인 및 접근 허용 여부 판단 수준의 인증·인가 기능
